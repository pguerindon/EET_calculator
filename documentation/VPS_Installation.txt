## Services

Application :

Fichier :
/etc/systemd/system/eet.service

Commandes :

sudo systemctl start eet
sudo systemctl stop eet
sudo systemctl restart eet
sudo systemctl status eet

Nginx :

sudo systemctl restart nginx
sudo systemctl status nginx

Fail2ban :

sudo fail2ban-client status
sudo fail2ban-client status sshd

## Configuration Nginx

Fichier :

/etc/nginx/sites-available/eet

Lien :

/etc/nginx/sites-enabled/eet

## Certificat HTTPS

Let's Encrypt

Obtention :

sudo certbot --nginx -d pg-chrono.fr -d [www.pg-chrono.fr](http://www.pg-chrono.fr)

Test renouvellement :

sudo certbot renew --dry-run

## Déploiement d'une nouvelle version

1. Générer le ZIP de production.

2. Déposer les fichiers ou le ZIP dans :

   /opt/eet_calculator

3. Mettre à jour les fichiers.

4. Redémarrer l'application :

   sudo systemctl restart eet

5. Vérifier :

   sudo systemctl status eet

## Règles de mise à jour

Modification d'un fichier Python :

* app.py
* config.py
* translation.py
* services/*.py

Action :

sudo systemctl restart eet

Modification d'un template HTML :

* templates/*.html

Action :

sudo systemctl restart eet

Modification d'un fichier statique :

* static/js/*.js
* static/css/*.css
* static/images/*
* static/*.ico

Action :

Aucun redémarrage nécessaire.

Mettre à jour la variable VERSION dans version.py afin de forcer le rechargement des ressources par les navigateurs.
